## What is Sector8?

Sector8 is a **deterministic execution governance layer** that sits as a control plane above enterprise LLM deployments. Every tool call an LLM makes passes through the Guard Module before reaching the OS, filesystem, or network.

The LLM is treated as an untrusted executor at all times. It proposes actions. The Guard Module decides whether they run.

## The core guarantee

<Info>
**LLM outputs are proposals, not commands.** No tool call executes without passing the Admission Controller — regardless of how the model was instructed, what context it received, or how many turns have elapsed.
</Info>

## Why pre-execution matters

Traditional AI security tools operate after the fact:

| Approach | When it runs | Data exposure |
|----------|-------------|---------------|
| **Sector8 Guard Module** | Before OS execution | None — data never accessed |
| Output filters / model safety | After LLM generates response | LLM has already seen data |
| SIEM / DLP | After network transmission | Data has already moved |
| Audit logs | After everything | Detective only — no prevention |
