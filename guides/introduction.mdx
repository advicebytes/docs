## What is Sector8?

Sector8 is a **deterministic execution governance layer** for enterprise AI. Every tool call an LLM agent makes passes through the Guard Module *before* it reaches your OS, filesystem, or network.

The LLM is treated as an **untrusted executor** at all times. It proposes actions. Sector8 decides whether they run.

## The core guarantee

<Info>
  **LLM outputs are proposals, not commands.** No tool call executes without passing the Admission Controller — regardless of how the model was instructed, what context it received, or how many turns have elapsed.
</Info>

## Why pre-execution matters

Most AI security tools operate after harm has already occurred. Sector8 is different.

| Approach | When it runs | Can it prevent harm? |
|---|---|---|
| **Sector8 Guard Module** | Before OS execution | **Yes — data never accessed** |
| Output filters / model safety | After LLM generates response | No — model has already seen data |
| SIEM / DLP | After network transmission | No — data has already moved |
| Audit logs | After everything | No — detective only |

## How it works

<Steps>
  <Step title="Agent proposes a tool call">
    Your LLM agent (Claude, GPT-4, Gemini) decides to call a tool — bash, file read, web fetch, database query.
  </Step>
  <Step title="Guard Module intercepts">
    The call is routed through Sector8's Admission Controller *before* execution. The policy engine evaluates it against your active invariants.
  </Step>
  <Step title="Decision with full audit trail">
    The Guard Module returns `ALLOW` or `DENY` with a cryptographic evidence hash, reason code, and decision trace ID — tamper-evident and forensically complete.
  </Step>
  <Step title="Safe execution or block">
    Allowed calls proceed. Denied calls are blocked with a forensic payload attached. Every decision is logged.
  </Step>
</Steps>
